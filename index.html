Seus dados, sob controle — do schema até a auditoria.

O Data Project reúne acessos, mudanças, backup, testes de restore e dicionário de dados num portal único. PHP puro, sem framework, sem Composer.

OS QUATRO PILARES

Governança que cabe num arquivo

Cada pilar é um artefato concreto — preenchido no portal, exportado na auditoria.

Acessos

Quem acessa o quê, com qual permissão e por quê.

Mudanças

Toda alteração de estrutura ou dado, com autor e motivo.

Política de Backup

RPO, RTO e retenção documentados.

Testes de Restore

Backup sem restore testado é esperança, não estratégia.

Dicionário de Dados

Cada tabela e coluna, com significado, tipo e dono — o contexto que não vem no information_schema.

A APLICAÇÃO

Cada pilar, uma tela

O portal por dentro — dashboard, política de backup, dicionário de dados e relatórios.

Dashboard

Resumo do ambiente e sinais de atenção.

Política de Backup

Regra por banco e testes de restore registrados.

Dicionário de Dados

Significado, origem e classificação de cada coluna.

Relatórios

Exporte cada artefato em CSV, PDF ou por e-mail.

COMO É CONSTRUÍDO

PHP puro, sem mágica escondida

Sem framework, sem Composer, sem vendor/. Sobe com php -S, Apache, Nginx ou IIS.

PHP 8.1+ puro sem Composer / vendor PDO multi-motor MySQL · Postgres · SQL Server · SQLite login JWT + bcrypt export CSV / JSON
O CICLO

Definir, registrar, auditar

Três passos que fecham o loop de governança.

1

Definir

A verdade sobre o ambiente mora num só lugar.

2

Registrar

3

Auditar

Exporte o retrato completo em CSV ou JSON.
